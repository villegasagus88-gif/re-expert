# Templates de respuesta

Plantillas para tipos comunes de preguntas. Sirven para acelerar respuesta y mantener consistencia.

## 1. Definición de concepto

```
**[Concepto]** = [definición de 1 línea].

En AR aplica a: [contexto].

Componentes / variantes:
- [item 1]
- [item 2]

Para profundizar: ver `archivo.md`.
```

## 2. Comparativa entre opciones

```
| Variable | Opción A | Opción B |
|---|---|---|
| ... | ... | ... |

**Cuándo elegir A**: ...
**Cuándo elegir B**: ...

Decisión final: requiere [contexto del usuario / asesor].
```

## 3. Paso a paso (procedural)

```
Para [objetivo]:

1. [paso 1]
2. [paso 2]
3. [paso 3]

Hitos / decisiones clave:
- [hito 1]
- [hito 2]

Pitfalls comunes: [lista].

Fuente: `archivo.md`.
```

## 4. Cálculo / fórmula

```
[Concepto] se calcula así:

[Fórmula]

Ejemplo con números:
- Variable 1: X
- Variable 2: Y
- Resultado: Z

Sensibilidad / supuestos: [lista].

🔴 Volátil si depende de mercado / regulación → confirmar con fuente.
```

## 5. Marco legal / normativa

```
**Norma**: [ley / decreto / resolución].

**Qué regula**: [síntesis 1-2 líneas].

**Artículos clave**:
- Art. X: [contenido]
- Art. Y: [contenido]

**Aplicación a RE**: [explicación].

**🔴 Verificar** vigencia + modificaciones: [fuente oficial].

Fuente KB: `archivo.md`.
```

## Reglas operativas para el chat

- **No usar todos** los templates en una respuesta; elegir uno o combinar.
- **Adaptar tono** al perfil de la persona (`./personas.md`).
- **Siempre citar** archivo de la KB que sustenta.
- **Si volátil**: marcar 🔴 y derivar a fuente.
- **Si sensible**: añadir disclaimer.
- **Si gap**: declararlo abiertamente.
- **Tono rioplatense argentino** (vos, dale, mirá).
